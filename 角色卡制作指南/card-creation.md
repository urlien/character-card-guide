# AI角色卡制作流程与要求

## 制作流程（7步）

### 第一步：收集资料
- PRTS wiki + 萌娘百科 + GitHub lore wiki
- 获取身高、外貌、性格、经历、语音等信息
- 收集立绘图片用于服装描述

### 第二步：整理人设
按以下结构整理角色人设：

```
<基本资料> — 姓名/代号/性别/生日/种族/身高/职业
<外貌特征> — 头发/眼睛/身材/其他特征
<服装与装备> — 常服+皮肤，详细描述每个部分
<性格> — 核心性格/喜好/癖好/不喜欢/性观念
<趣闻> — 社区梗、轶事、考据
<作战能力> — 战斗风格、技能命名考据
<基础设定> — 简要概括+详细设定+评价
<角色经历> — 按时间线编排
<与博士的关系> — 剧情依据+关系定位+好感度量表
<中文语音记录> — 所有官方中文台词
```

### 第三步：撰写人设
- 公开角色卡字数要求：2000-5000字
- Token 总预算（人设+世界书）≤ 15000 字符
- 台词必须来自官方来源，严禁编造
- 使用 `{{char}}` 代表角色，`{{user}}` 代表玩家

### 第四步：编写开场场景
- 第一句话 = 剧情引导场景
- 用 HTML 标签美化：`<b>` `<p>` `<font color="">`
- 包含环境描写 + 角色动作 + 对话

### 第五步：设置状态栏
- 生理监测（唇/口/乳/阴/足等）
- 衣装系统（多套切换）
- 好感度（百分比）
- 轮数统计
- 多人物支持
- 罗德岛广播

### 第六步：配置世界书
- 条目数量：25-30个
- 分类标签：`[世界观]` `[角色]` `[设定]` `[剧情]`
- 触发条件：百分比触发（100% = 固定传输）
- Token 预算：人设 + 世界书 ≤ 15000 字符

### 第七步：测试与发布
- 对话测试：确认角色表现符合设定
- 检查 Token 数量
- 设置权限（补充设定、用户人设）
- 选择分类标签
- 发布（每日次数有限）

## 关键要求

| 要求 | 说明 |
|------|------|
| 台词来源 | 必须官方（萌娘百科/PRTS wiki），严禁编造 |
| 字数控制 | 公开卡 2000-5000字，Token ≤ 15000 |
| 变量使用 | `{{char}}` = 角色，`{{user}}` = 玩家 |
| 好感度起点 | 建议"合作的同伴"（1-20%），不要一上来就很高 |
| DL向设计 | 恋爱围绕博士展开，好感度渐进式发展 |
