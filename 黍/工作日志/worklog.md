- API 基础信息：
  - 域名：`https://www.sexyai.ai`
  - 认证方式：Authorization Bearer token + rptoken (JWT cookie)
  - rptoken 过期时间：2026-07-16
  - 所有请求均为 POST 方法，需同时带 token 和 rptoken
  - 需带 Header `lang: ZH`
- 已掌握的 API 端点：
  - `POST /api/role/query` — 查询任意角色卡详情（body: `{"id": 卡ID}`）
  - `POST /api/role/lorebooks/fetch` — 查询世界书元数据（body: `{"roleId": 卡ID}`）
  - `POST /api/role/customize/list` — 查询自己的卡列表
  - `POST /api/role/comment/page` — 查询评论
  - `POST /api/user/info` — 查询用户信息
- **注意：** 世界书条目内容 API 未返回（`/api/role/lorebooks/fetch` 只返回元数据），实际条目内容需要网页端登录后查看

#### writing-style-skill v2.0 核心内容

已将 jiangnan.skill 的叙事技法引擎完全融入：

**DNA 规则：**
- DNA-1 缺失驱动：情感引擎 = 失去/即将失去
- DNA-4 反差构图：宏大背景 × 微小细节（每 500-800 字至少一次）
- DNA-5 蓄力-释放节奏：开头平静 → 中段蓄力 → 结尾释放或留白
- DNA-6 通感激活：每段至少一个感官细节

**5 个必用句式：**
- ★1 反转对比："{宏大}又怎样？{渺小}才是{真相}"
- ★2 预设怀念："那时候{她}还不知道，{遗憾}"
- ★3 软转折："{硬话}{哪怕/可是/不过/只是}{柔话}"
- 4 物哀升华："就像{寻常物事}，{不寻常的含义}"
- 5 时空折叠："仿佛{过去}重叠在了{当下}"

**信物系统：** 用具体物件承载情感（如钓鳞绳线、节气歌、泥土）
**不完整闭合：** 最后一句是动作/画面，不总结、不升华
**去 AI 味：** 51 条减法规则 + 6 条打磨规则

**角色卡叙事结构模板：**
```
第一幕（30%）：单人镜头 — 角色独自做日常小事，感官描写建立氛围
第二幕（20%）：广角镜头 — 镜头拉远，展示环境全貌
第三幕（35%）：角色互动 — 第二个角色登场，对话推动性格展现
第四幕（15%）：收束 — 留白，不总结
```
