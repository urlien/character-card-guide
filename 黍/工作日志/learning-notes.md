# 学习心得 — Discord 技术库研读

## 一、状态栏技术方案总结

### 1. 基础 Markdown 状态栏（入门级）
- 使用三个反引号 ` ``` ` 包裹状态栏内容
- 放在第一句话中，刷新后可见
- 最简单但功能有限，无交互、无美化
- 适合新手快速上手

### 2. KV 数据块格式（DJ工坊方案）⭐ 推荐
**格式：** `LS▶key::value;;key::value;;◀LE`

**核心字段设计：**
- 系统字段：`dt`（日期时间）、`loc`（场景地点）、`int`（互动计数）、`sum`（情节概要）
- 角色字段（动态，c+数字前缀）：
  - `cXn` 角色名字
  - `cXa` 身份/关系/年龄
  - `cXl` 外貌与着装描述
  - `cXc` 当前动作与姿势
  - `cXf` 身体与心理感受
  - `cXimg` 角色头像图片URL
- 主角字段：`un`、`ua`、`ul`、`uc`、`uf`（每次必须显示）

**行动选项格式：**
`DGS_KV_TIPS_START▶tips:1.选项内容\n2.选项内容;◀DGS_KV_TIPS_END`

**优点：**
- 角色数量无上限，动态增减
- 支持角色头像图片
- 有成熟的正则配套
- 配色可调（紫粉色、黑灰色、纸张米白色等）

### 3. 正则四步法（澄清/柑硕方案）⭐ 最稳定
- 使用键值对格式，`::` 分隔键值，`;;` 分隔对
- 四步法流程：生成KV → 正则捕获 → 替换渲染 → 美化显示
- 不依赖传统的捕获式正则，更稳定
- 柑硕的"不会坏的状态栏结构正则"进一步增强了鲁棒性
- 即使炸了也不会太明显（空字段而非整个崩坏）

### 4. JavaScript 内联状态栏（交互型）
- 使用内联 JS 实现可点击按钮、页面切换等交互功能
- 需要配合正则维持跨消息状态
- 支持 Tab 页切换、选项点击等高级交互
- 手机端可能有兼容性问题

### 5. Web Components + Shadow DOM（GodCount 新标准）⭐ 最先进
- 使用 `<z-live-widget>` 等自定义标签
- Shadow DOM 实现样式隔离，不会被 MMD 剥离
- 性能最优：组件级隔离渲染，避免全局污染
- 支持 `@media` 区分手机端削减动画
- 模块化设计，只暴露接口
- 缺点：需要一定前端基础

### 6. 侧边栏按钮模块（Tony 方案）
- 固定在页面右侧的悬浮按钮
- 点击自动将预设文字填入输入框
- HTML + CSS + JS 三件套
- `data-t` 属性存储按钮文本
- `position: fixed; right: 0; z-index: 1000;`
- 适合放防神化指令、总结模板、游玩指令等

### 7. Galgame 式对话框（栖夜方案）
- 模拟视觉小说的对话框样式
- 需要在字段前加 `[gal]` 标记
- 底部有可点击的选项按钮
- 已做手机版适配

## 二、正则相关要点

- **键值对格式**是目前主流：`key::value` 用 `;;` 分隔
- **不要使用全角符号**替代 `::` 和 `;;`
- 数据块内**禁止换行**
- 每对末尾都要有 `;;`
- AI 生成时需要在角色设定中明确格式要求
- 重要信息放在角色设定的**前面**，AI 会优先读取

---

## 三、美化相关

- 全局美化通过正则替换实现
- 支持日间/夜间双模式兼容
- 可通过侧边栏按钮切换主题
- CSS 动画是性能开销大头，手机端需注意

## 五、参考卡深度分析 — #221817「明日方舟·泰拉X塔卫二」

### 卡片概况
- 作者：伊纯 | 使用量：173万 | 玩家数：10,867 | 评分：10.0
- 世界书：528条条目 / 255,571关键词（~25万字）
- 人设字数：4,012字

### 状态栏架构（标签锚定式）
这张卡使用了非常成熟的**标签锚定式正则架构**，约30+个正则模块：

| 标签 | 功能 |
|------|------|
| `<m1>`~`<m4>` | 主面板模块 |
| `<指令1>` `<指令2>` | 指令/操作模块 |
| `<库1>` | 资料库模块 |
| `<左侧1>`~`<左侧6>` | 左侧面板6个tab |
| `地图1`~`地图3` | 地图模块 |
| `tj1`~`tjd` | 统计数据模块（13个子模块） |
| `<c1>`~`<c9>` `<ca>` `<cb>` | 角色面板（11个slot） |

### 开场白设计
- 使用 `全局美化1` `全局美化2` `全局美化3` 等标记触发正则美化
- 正文用 HTML 标签做精美排版（`<p>`/`<span>`/`<b>`/`<i>` 带颜色、背景、边框）
- 末尾有 `选人1`~`选人4` 角色选择面板

### 对黍卡的启示
1. **标签锚定式 vs KV 数据块**：#221817 用标签锚定式（每个模块一个触发标记），比纯 KV 格式更灵活，适合多模块复杂面板
2. **HTML 美化开场白**：视觉效果好，黍卡可以参考这种风格
3. **模块化设计**：角色、地图、统计、指令分离成独立模块，便于维护
4. **世界书规模**：528条/25万字，说明深度世设卡的世界书可以做得很大
5. **状态栏与正则的关系**：状态栏标记（如 `<m1>`）是正则的触发锚点，AI 输出标记 → 正则捕获 → 替换为 HTML/CSS/JS 可视化面板

### sexyai.ai API 使用记录
- 域名：`https://www.sexyai.ai`
- 认证：Authorization Bearer token + rptoken (JWT cookie)
- 已用端点：
  - `POST /api/role/query` — 成功获取卡片详情
  - `POST /api/role/lorebooks/fetch` — 成功获取世界书元数据（但条目内容未返回）
- 世界书条目内容需要网页端登录后查看，API 只返回元数据

### 关键技术要点

**江南技法 DNA 规则（已融入 writing-style-skill）：**
- DNA-1 缺失驱动：情感引擎 = 失去/即将失去
- DNA-4 反差构图：宏大 × 微小（每 500-800 字至少一次）
- DNA-5 蓄力-释放节奏：平静 → 蓄力 → 释放/留白
- DNA-6 通感激活：每段至少一个感官细节

**5 个必用句式：**
- ★1 反转对比："{宏大}又怎样？{渺小}才是{真相}"
- ★2 预设怀念："那时候{她}还不知道，{遗憾}"
- ★3 软转折："{硬话}{柔转折词}{柔话}"
- 4 物哀升华："就像{寻常物事}，{不寻常的含义}"
- 5 时空折叠："仿佛{过去}重叠在了{当下}"
