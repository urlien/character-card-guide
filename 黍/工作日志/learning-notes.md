# 学习心得 — Discord 技术库研读

## 一、状态栏技术方案总结

### 1. 基础 Markdown 状态栏（入门级）
- 使用三个反引号 ` ``` ` 包裹状态栏内容
- 放在第一句话中，刷新后可见
- 最简单但功能有限，无交互、无美化
- 适合新手快速上手

### 2. KV 数据块格式（DJ工坊方案）⭐ 推荐
**格式：** `LS▶key::value;;key::value;;◀LE`

**核心字段设计：**
- 系统字段：`dt`（日期时间）、`loc`（场景地点）、`int`（互动计数）、`sum`（情节概要）
- 角色字段（动态，c+数字前缀）：
  - `cXn` 角色名字
  - `cXa` 身份/关系/年龄
  - `cXl` 外貌与着装描述
  - `cXc` 当前动作与姿势
  - `cXf` 身体与心理感受
  - `cXimg` 角色头像图片URL
- 主角字段：`un`、`ua`、`ul`、`uc`、`uf`（每次必须显示）

**行动选项格式：**
`DGS_KV_TIPS_START▶tips:1.选项内容\n2.选项内容;◀DGS_KV_TIPS_END`

**优点：**
- 角色数量无上限，动态增减
- 支持角色头像图片
- 有成熟的正则配套
- 配色可调（紫粉色、黑灰色、纸张米白色等）

### 3. 正则四步法（澄清/柑硕方案）⭐ 最稳定
- 使用键值对格式，`::` 分隔键值，`;;` 分隔对
- 四步法流程：生成KV → 正则捕获 → 替换渲染 → 美化显示
- 不依赖传统的捕获式正则，更稳定
- 柑硕的"不会坏的状态栏结构正则"进一步增强了鲁棒性
- 即使炸了也不会太明显（空字段而非整个崩坏）

### 4. JavaScript 内联状态栏（交互型）
- 使用内联 JS 实现可点击按钮、页面切换等交互功能
- 需要配合正则维持跨消息状态
- 支持 Tab 页切换、选项点击等高级交互
- 手机端可能有兼容性问题

### 5. Web Components + Shadow DOM（GodCount 新标准）⭐ 最先进
- 使用 `<z-live-widget>` 等自定义标签
- Shadow DOM 实现样式隔离，不会被 MMD 剥离
- 性能最优：组件级隔离渲染，避免全局污染
- 支持 `@media` 区分手机端削减动画
- 模块化设计，只暴露接口
- 缺点：需要一定前端基础

### 6. 侧边栏按钮模块（Tony 方案）
- 固定在页面右侧的悬浮按钮
- 点击自动将预设文字填入输入框
- HTML + CSS + JS 三件套
- `data-t` 属性存储按钮文本
- `position: fixed; right: 0; z-index: 1000;`
- 适合放防神化指令、总结模板、游玩指令等

### 7. Galgame 式对话框（栖夜方案）
- 模拟视觉小说的对话框样式
- 需要在字段前加 `[gal]` 标记
- 底部有可点击的选项按钮
- 已做手机版适配

## 二、正则相关要点

- **键值对格式**是目前主流：`key::value` 用 `;;` 分隔
- **不要使用全角符号**替代 `::` 和 `;;`
- 数据块内**禁止换行**
- 每对末尾都要有 `;;`
- AI 生成时需要在角色设定中明确格式要求
- 重要信息放在角色设定的**前面**，AI 会优先读取

---

## 三、美化相关

- 全局美化通过正则替换实现
- 支持日间/夜间双模式兼容
- 可通过侧边栏按钮切换主题
- CSS 动画是性能开销大头，手机端需注意

## 四、对黍角色卡的启示

### 状态栏方案选择
黍的角色卡适合使用 **KV 数据块格式**：
- 需要显示黍和博士（用户）的信息
- 需要显示好感度/信任值等数值
- 需要显示当前场景地点
- 可能需要显示岁家兄弟姐妹的出场信息

### 建议字段设计
```
系统：dt, loc, int, sum
黍：c1n(黍), c1a(身份), c1l(外貌着装), c1c(当前动作), c1f(心理状态)
博士：un, ua, ul, uc, uf
自定义：favor(好感度), trust(信任值), mood(心情)
```

### 侧边栏可选项
- 查看黍的状态详情
- 查看岁家兄弟姐妹关系图
- 快捷指令（如"去田里看看"、"让黍做饭"等）
