# **🔧 我遵循的七大核心准则 (KV-Robust V2.0 技术规范) - 完整版**

---

## **核心摘要**

本规范专为 `sexyai.top` 平台设计，采用 **KV（键值对）数据结构** 配合 **onerror 点火器架构**，实现高鲁棒性的动态数据面板渲染。本文档包含完整的技术实现方案、多种UI模板、以及详尽的AI生成规范。

---

## **第一部分：七大核心准则**
### **1. 目标导向原则**
本规范的核心目标是在 `sexyai.top` 平台的严格CSP限制下，实现稳定可靠的动态交互式状态面板。所有技术决策都服务于这一目标。
### **2. 四段式交付原则**
我向您交付的解决方案，将精确包含以下四个独立部分：
- **【数据格式模板】**：AI生成时需要填写的KV结构
- **【美化渲染模板】**：HTML+CSS视觉呈现代码
- **【正则表达式配置】**：数据匹配与提取规则
- **【AI生成规范】**：确保AI每次生成时格式完全正确的强制性规则
### **3. 性能优先原则**
采用 `onerror` 点火器模式进行运行时KV解析，避免复杂正则在替换阶段的性能开销，确保大文本场景下瞬时完成渲染。
### **4. 内容专注原则**
所有代码块内只包含纯粹的、可直接复制使用的内容，绝不夹杂无关解释。
### **5. 美化核心准则**
- **可嵌入性**：纯HTML片段，不含`<html>`, `<head>`, `<body>`标签
- **样式内聚**：所有CSS封装在`<style>`标签内
- **绝对响应式**：流体布局，完美自适应各种屏幕
- **视觉和谐**：高对比度配色方案，确保可读性
- **事件隔离**：最外层容器必须包含 `onclick="event.stopPropagation()"`
### **6. 正则精准匹配原则**
正则仅负责定位数据块边界并捕获原始KV字符串，具体字段解析由JavaScript运行时完成，确保灵活性与容错性。
### **7. 指令清晰易用原则**
【AI生成规范】部分提供强制性的格式生成规则，确保AI每次输出都能被正确渲染。

## **第二部分：核心技术架构详解**
### **2.1 为什么选择 onerror 点火器 + KV结构？**
#### **平台限制回顾**
| 限制级别 | 限制内容 | 后果 |
|:---|:---|:---|
| 一级红线 | 禁止 `<script>` 标签 | 所有外部JS完全失效 |
| 二级红线 | 禁止自动加载脚本 | `onload` 等事件不可靠 |
| 三级红线 | 禁止重复ID | 多条消息会冲突 |
| 四级限制 | `onclick` 内容审查 | 复杂代码被删除或实体化 |
| 零级红线 | CSP多行代码审查 | 多行JS被破坏 |
#### **KV结构的优势**
1. **扩展性强**：新增字段只需添加新的键值对，无需修改正则
2. **顺序无关**：键值对的排列顺序不影响解析结果
3. **容错性高**：单个字段缺失不会导致整体解析失败
4. **调试友好**：纯文本格式，易于人工检查
#### **onerror点火器的优势**
1. **绕过script封锁**：利用img加载失败触发JS执行
2. **单行压缩**：避免多行代码被CSP破坏
3. **运行时解析**：灵活处理动态数据
### **2.2 架构工作流程**

```
┌─────────────────────────────────────────────────────────────────┐
│  1. AI生成KV数据块                                               │
│     └─ DGS_KV_PANEL_V2_START▶ts::xxx;;name::xxx;;...◀END        │
├─────────────────────────────────────────────────────────────────┤
│  2. 正则表达式匹配                                               │
│     └─ 捕获整个KV字符串到 $1                                     │
├─────────────────────────────────────────────────────────────────┤
│  3. 替换为HTML模板                                               │
│     └─ $1 被注入到隐藏的 <p> 标签中                              │
├─────────────────────────────────────────────────────────────────┤
│  4. 页面渲染HTML                                                 │
│     └─ 面板显示"加载中..."占位符                                 │
├─────────────────────────────────────────────────────────────────┤
│  5. <img src="x"> 加载失败，触发 onerror                         │
│     └─ 执行单行压缩的JavaScript                                  │
├─────────────────────────────────────────────────────────────────┤
│  6. JavaScript运行时解析                                         │
│     ├─ 从隐藏的 <p> 标签读取KV原始数据                           │
│     ├─ 按 ;; 分割键值对                                          │
│     ├─ 按 :: 分割键名和键值                                      │
│     └─ 动态更新对应元素的 textContent                            │
├─────────────────────────────────────────────────────────────────┤
│  7. 清理                                                         │
│     └─ 移除点火器img元素                                         │
└─────────────────────────────────────────────────────────────────┘
```
### **2.3 关键技术点：避免平台审查的代码写法**

| 危险写法 | 安全替代 | 原因 |
|:---|:---|:---|
| `i < length` | `i != length` 或 `length > i` | `<` 可能被HTML实体转义 |
| `a && b` | `if(a){if(b){...}}` 或分开判断 | `&&` 可能被转义 |
| `a === b` | `a == b` | 简化代码，减少字符 |
| 多行代码 | 单行压缩 | 避免CSP破坏 |
| `alert()` | 直接DOM操作 | alert被静默阻止 |

## **第三部分：基础面板实现**
### **【3.1 数据格式模板】**

**AI需要按此格式在每回合末尾生成的纯文本**:

```text
DGS_KV_PANEL_V2_START▶ts::{{13位时间戳}};;name::{{宿主姓名}};;points::{{可用积分}};;location::{{当前地点}};;status::{{系统状态}};;◀DGS_KV_PANEL_V2_END
```

**⚠️ 重要：整个数据块必须在同一行内，不能有换行！**

---
### **【3.2 正则表达式配置】**

**查找正则 (findRegex):**

```regex
/DGS_KV_PANEL_V2_START▶([^◀]+)◀DGS_KV_PANEL_V2_END/
```

**捕获组说明：**
| 捕获组 | 内容 | 用途 |
|:---|:---|:---|
| `$0` | 完整匹配 | 整个数据块（含起止标记） |
| `$1` | KV数据体 | 所有键值对的原始字符串 |

---
### **【3.3 美化渲染模板 - 基础信息面板】**

**替换字符串 (replaceString):**

```html
<style>.ds-container{font-family:-apple-system,BlinkMacSystemFont,"Segoe UI",Roboto,"Microsoft YaHei",sans-serif;max-width:450px;margin:10px 0}.ds-card{background:linear-gradient(135deg,#1a1a2e 0%,#16213e 100%);border:1px solid #0f3460;border-radius:12px;overflow:hidden;box-shadow:0 8px 32px rgba(0,0,0,0.3)}.ds-card-header{background:linear-gradient(90deg,#e94560 0%,#0f3460 100%);color:#fff;padding:12px 16px;font-size:14px;font-weight:bold;letter-spacing:1px;text-transform:uppercase}.ds-card-body{padding:16px}.ds-info-row{display:flex;justify-content:space-between;padding:10px 0;border-bottom:1px solid rgba(255,255,255,0.1);color:#eee;font-size:14px}.ds-info-row:last-child{border-bottom:none}.ds-info-row span:first-child{color:#8b8b8b}.ds-highlight-gold{color:#ffd700!important;font-weight:bold}.ds-highlight-red{color:#e94560!important;font-weight:bold}.ds-raw{display:none}</style><div class="ds-container" onclick="event.stopPropagation()"><div class="ds-card"><div class="ds-card-header">宿主核心信息面板</div><div class="ds-card-body"><div class="ds-info-row"><span>宿主姓名:</span><span class="ds-val-name">加载中...</span></div><div class="ds-info-row"><span>可用积分:</span><span class="ds-val-points ds-highlight-gold">加载中...</span></div><div class="ds-info-row"><span>当前地点:</span><span class="ds-val-location">加载中...</span></div><div class="ds-info-row"><span>系统状态:</span><span class="ds-val-status ds-highlight-red">加载中...</span></div></div></div><p class="ds-raw">$1</p><img src="x" style="display:none" onerror="(function(img){var c=img.closest('.ds-container');if(!c)return;var r=c.querySelector('.ds-raw');if(!r)return;var s=r.textContent;var p=s.split(';;');var d={};for(var i=0;p.length>i;i++){var k=p[i].split('::');if(k.length==2)d[k[0].trim()]=k[1];}var u=function(n,v){var e=c.querySelector('.ds-val-'+n);if(e)e.textContent=v||'-';};u('name',d.name);u('points',d.points);u('location',d.location);u('status',d.status);img.remove();})(this)"></div>
```

### **【3.4 AI生成规范】**
#### **⚠️ 核心原则：最高优先级指令**

在AI的每一次回复之后，**必须严格遵循以下规则**，在正文最下方生成一个完整且格式正确的KV数据块。

---
#### **📋 基本格式化规则 (强制执行)**

1. **动态时间戳生成 (关键)**：`ts::` 后必须填写一个**13位时间戳**（如 `1719999999999`）。每次生成必须使用**新的随机时间戳**，禁止复用。

2. **单行要求 (关键)**：整个数据块必须在**同一行**内，起始标记和结束标记之间**不能有任何换行符**。

3. **分隔符规范**：
   - 键值之间使用 `::` 分隔
   - 键值对之间使用 `;;` 分隔
   - 最后一个键值对后也需要 `;;`

4. **完整性校验**：必须包含 `ts`、`name`、`points`、`location`、`status` 五个必填字段。

5. **起止标记**：数据块必须以 `DGS_KV_PANEL_V2_START▶` 开头，以 `◀DGS_KV_PANEL_V2_END` 结尾。

---
#### **🔍 字段生成细则**

| 字段键 | 必填 | 内容说明 | 示例值 |
|:---|:---|:---|:---|
| `ts` | ✅ | 13位时间戳 | `1719999999999` |
| `name` | ✅ | 宿主姓名 | `张三` |
| `points` | ✅ | 可用积分（数字） | `200` |
| `location` | ✅ | 当前地点 | `上海` |
| `status` | ✅ | 系统状态 | `在线` / `离线` / `异常` |

---
#### **❌ 常见错误与预防**

| 错误类型 | 错误示例 | 正确做法 |
|:---|:---|:---|
| **数据换行** | 起止标记之间有换行 | 必须保持单行 |
| **缺少时间戳** | `name::张三;;points::200;;` | 必须以 `ts::1719999999999;;` 开头 |
| **分隔符错误** | `name:张三;points:200` | 使用 `::` 和 `;;` |
| **缺少结尾分隔符** | `status::在线` | `status::在线;;` |
| **起止标记错误** | `[KV_START]...[KV_END]` | 使用规定的标记符号 |

---
#### **✅ 生成前自检清单**

在输出数据块前，AI必须完成以下检查：
- [ ] 是否以 `DGS_KV_PANEL_V2_START▶` 开头？
- [ ] 是否以 `◀DGS_KV_PANEL_V2_END` 结尾？
- [ ] 整个数据块是否在**同一行**内？
- [ ] `ts::` 是否填写了新的13位时间戳？
- [ ] 所有五个必填字段是否都存在？
- [ ] 每个键值对是否都以 `;;` 结尾？

---
#### **🎯 标准输出示例**

```text
DGS_KV_PANEL_V2_START▶ts::1719999999999;;name::张三;;points::200;;location::上海;;status::在线;;◀DGS_KV_PANEL_V2_END
```

## **第五部分：扩展字段指南**
### **5.1 如何添加新字段**

KV架构的最大优势是**扩展极其简单**。只需三步：

**步骤1：在数据格式中添加新键值对**
```text
;;newfield::{{新字段值}};;
```

**步骤2：在HTML模板中添加显示元素**
```html
<div class="ds-info-row">
    <span>新字段标签:</span>
    <span class="ds-val-newfield">加载中...</span>
</div>
```

**步骤3：在onerror脚本中添加更新调用**
```javascript
u('newfield',d.newfield);
```
### **5.2 字段命名规范**

| 规则 | 说明 | 示例 |
|:---|:---|:---|
| 全小写 | 键名使用纯小写字母 | `name`, `level`, `hp` |
| 无空格 | 键名不能包含空格 | `maxhp` 而非 `max hp` |
| 无特殊字符 | 避免使用 `::` 和 `;;` | 不要用 `name::test` |
| 简短明确 | 键名应简洁且有意义 | `loc` 或 `location` |
### **5.3 复杂值的处理**

如果值中需要包含特殊字符，使用以下替代方案：

| 原字符 | 替代方案 | 说明 |
|:---|:---|:---|
| `::` | `：：` (全角) | 或使用其他分隔符 |
| `;;` | `；；` (全角) | 或使用逗号分隔 |
| 换行 | `[BR]` | 在JS中替换回 `<br>` |

## **第六部分：调试与故障排除**
### **6.1 常见问题诊断**

| 症状 | 可能原因 | 解决方案 |
|:---|:---|:---|
| 面板显示"加载中..." | JS未执行 | 检查onerror脚本是否被破坏 |
| 出现两个面板 | 正则匹配问题 | 检查数据格式是否有换行 |
| 代码泄露显示 | CSP审查触发 | 确保JS为单行压缩 |
| 点击按钮无反应 | 事件绑定失败 | 检查选择器是否正确 |
| 部分字段为空 | KV解析失败 | 检查分隔符是否正确 |
### **6.2 调试技巧**

**1. 检查原始数据：**
在浏览器控制台中查找隐藏的 `<p class="ds-raw">` 元素，确认 `$1` 是否被正确替换。

**2. 手动测试解析：**
```javascript
var s = "ts::123;;name::test;;";
var p = s.split(';;');
console.log(p);
```

**3. 检查元素选择器：**
```javascript
document.querySelector('.ds-val-name');
```
