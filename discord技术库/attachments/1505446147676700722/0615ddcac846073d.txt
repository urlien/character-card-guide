通用状态栏开发规范（抓取式 / 键值对 / 三段正则架构）

---

零、核心设计理念

数据与 UI 分离

· AI 只输出干净的键值对数据（[键名=值]）
· 正则引擎负责将数据注入 HTML 框架并渲染
· 用户永远看不到原始数据，只能看到卡片

三段正则分工

正则 位置 触发标签 作用
第一段 局部（用户人设/角色卡） <UI> 或自定义标签 将触发标签替换为带 JS 解析引擎的 HTML 框架（img.onerror 自执行）
第二段 局部（紧接第一段之后） [键=值] 正则 将所有键值对包裹进隐藏标签 display:none，防止原文字显示
第三段 全局（功能栏） <CSS> 将样式触发标签替换为完整 <style> 代码块，全局生效

为什么 CSS 必须放全局功能栏？

· 如果放局部正则，AI 每次输出都会带 <CSS>，可能被后续对话的替换逻辑干扰
· 全局功能栏的替换只执行一次，样式持久生效，且不会被后续消息覆盖

---

一、AI 输出格式规范（写入角色卡 / 用户人设）

【状态栏输出铁律】

每次回复末尾，你必须且只能输出以下格式的数据。不要添加任何其他文字、符号、空行、分割线或说明。

<UI_UI>
[时间=当前时间]
[地点=当前地点]
[角色1类型=类型]
[角色1姓名=名字]
[角色1属性1=值]
[角色1属性2=值]
[选项1=行动选项1]
[选项2=行动选项2]

规则：
1. 必须以单独一行 <UI_UI> 开头。
2. 所有数据必须用 [键=值] 格式，每行一个。
3. 键名和值中不要出现方括号、等号、分号。
4. 多个角色用数字编号递增：[角色1姓名=xxx]、[角色2姓名=xxx]。
5. 结尾不要添加 <CSS> 或任何其他标签。

---

二、正则配置

第一段（局部 - 放用户人设/角色卡第一条）

匹配： <UI_UI>

替换为： 带 JS 解析引擎的 HTML 框架

框架核心逻辑（写在 img.onerror 中）：

1. 定位当前消息块（compareDocumentPosition 避免跨消息污染）
2. 扫描 DOM 中所有 [键=值] 文本（包括被第二段隐藏的 span）
3. 解析成 kv 对象：{ "角色1姓名": "阿尔斯", "角色1能量": "32%", ... }
4. 根据角色类型动态生成对应卡片（createElement 构建 DOM）
5. 将生成的 DOM 插入到当前消息下方
6. 设置定时器定期检查元素是否还在 DOM 中

替换内容关键点：

· 所有 JS 必须内联在 onerror 属性中（不能用 <script> 标签）
· 用 document.createElement 动态生成 DOM，不能用 innerHTML（防 XSS 过滤）
· 必须停止事件冒泡 event.stopPropagation()
· 选项按钮需自动填入输入框

---

第二段（局部 - 放用户人设/角色卡第二条）

匹配： \[([^=\[\]]+)=([^\[\]]+)\]\s*

替换为： <span style="display:none">[$1=$2]</span>

作用： 将 AI 输出的 [时间=2024年] 这类原文字全部隐藏，用户看不到。

注意： 这条正则必须在第一段之后执行，否则框架代码读不到数据。

---

第三段（全局 - 放功能栏）

匹配： <CSS>

替换为： 完整的 <style>...</style> 代码块

为什么放全局？

· 样式只需注入一次，全局生效
· 不会被后续对话的替换逻辑影响
· 减少 AI 每次输出的 token 消耗

样式设计规范：

· 所有颜色用 CSS 变量定义在 :root 中，方便换肤
· 卡片用 nx- 前缀命名，避免与平台样式冲突
· 展开/收起用 checkbox hack 实现（:checked ~ .target）
· 过渡动画用 max-height + transition，不用 JS

---

三、数据流完整流程

AI 输出 → 第一段正则匹配 → 开始构建框架
[时间=2024年] → 第二段正则隐藏原文字 → (原文字消失)
[角色1姓名=阿尔斯] → 框架JS读取隐藏span数据 → 解析为kv对象
框架JS生成DOM卡片 → 卡片插入页面
第三段正则（全局）注入CSS → 卡片有样式

---

四、常见问题及解决方案

问题 原因 解决
纯文本键值对裸露 第二段正则没执行或顺序错误 确保第二段在第一段之后
卡片没有样式 <CSS> 没有被全局正则替换 检查全局正则是否启用
多轮对话后卡片消失 元素被平台回收 框架内的 setInterval 定时检查并重新渲染
跨消息数据污染 扫描范围过大 用 compareDocumentPosition 精确定位当前消息
选项按钮点击无效 输入框选择器不对 用多个选择器兜底：textarea, .uni-textarea-textarea 等

五、框架 JS 核心代码模板（简化版）

// 定位当前消息块
var box = e.parentNode;
if (box && (box.tagName === 'P' || box.tagName === 'FONT')) box = box.parentNode;

// 正则匹配所有 [键=值]
var mR = /\[([^=\[\]]+)=([^\[\]]+)\]/g;
var kv = {};

// 扫描 DOM 中的隐藏键值对
var els = document.querySelectorAll('.item, .content, .vditor-reset, .vditor-preview');
for (var i = 0; i < els.length; i++) {
  if (box.compareDocumentPosition(els[i]) & 2) {
    // 在当前消息范围内，读取数据
    var m;
    while ((m = mR.exec(els[i].textContent)) !== null) {
      kv[m[1].trim()] = m[2].trim();
    }
  }
}

六、给其他 AI 的复制用模板

请根据以下规范为我写一个状态栏：

1. 架构：三段正则 + 键值对抓取
2. AI输出格式：以 <UI> 开头，每行一个 [键=值]，结尾不加 <CSS>
3. 第一段正则：匹配 <UI>，替换为 img.onerror 的JS框架（动态生成DOM）
4. 第二段正则：匹配 [键=值]，替换为 <span style="display:none">
5. 第三段正则：匹配 <CSS>，替换为 <style> 样式，放在全局功能栏
6. 主题：红银配色（奥特曼风格）
7. 字段：时间、地点、角色（支持多角色，按类型渲染不同卡片）、选项按钮

要求：
- 不用 <script> 标签，所有JS内联在 onerror 中
- 用 createElement 构建DOM，不用 innerHTML
- 角色类型通过 [角色N类型=xxx] 区分，框架自适应渲染
- 样式用 CSS 变量定义颜色，方便换肤
