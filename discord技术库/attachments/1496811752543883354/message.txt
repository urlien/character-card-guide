角色卡违规处理与处罚细则
版本号：v1.5　生效日期：2026年5月8日　最近修订：2026年5月8日

第一章　总则

1.1　适用范围
本细则适用于公开发布于魅魔网站平台任意站点 / 页面、可被用户访问与传播的角色卡。处理范围包括角色卡的全部内容，包括但不限于标题、简介、第一句话、设定、封面、分区、标签及其关联素材、美化内容等。

1.2　制定目的
为维护社区创作生态、作品曝光分发的公平性以及内容呈现的规范性，平台对各类破坏原创秩序、扰乱发布秩序、影响社区公平、违反内容合规要求或损害用户体验的违规行为进行规范处理。本细则旨在保障创作者合法权益，维护平台内容生态、创作秩序、发布秩序、内容合规与分发公平，减少违规行为对用户认知、游玩体验、社区环境及平台运营造成的混淆、误导与不良影响；同时，在兼顾既有用户体验与数据延续的前提下，鼓励作者通过合规创作与有效迭代持续提升作品质量。

1.3　术语定义
本细则中以下术语的含义如下：
- 平台：指魅魔网站及其运营管理主体，为本细则的制定与适用主体。
- 审核组：指平台授权负责本细则具体执行、举报核实、申诉受理及相关适用说明的工作团队。
- 将该卡设为私密：将角色卡转为私密状态，不再对外公开展示，亦不进入公开访问范围；已开始游玩的用户不受影响，可继续游玩。私密期间角色卡不再产生收益电量；作者仍可通过主动分享私密链接供他人访问体验。因违规处理导致角色卡被设为私密期间所产生的榜单排名变动、流量损失及曝光中断，由作者自行承担，平台不予补偿；但经申诉确认属于平台判定错误的除外。
- 封禁投递权限：账号失去投稿 / 公开投递权限；其他功能不受影响，除非本细则另有说明。
- 扣除收益电量：扣除该角色卡截至处理生效时点已经获得的收益电量。除本细则另有明确说明外，扣除范围以该违规角色卡自身已获得的收益电量为限；若涉及多张角色卡或多个账号共同违规，则按对应条款对相关角色卡分别处理。本细则中"扣除全部收益电量""扣除 X% 收益电量"等表述，均以本定义为基准计算。
- 去旧留新：当新版本被认定为"有效更新"时，平台保留新版公开，并将旧版设为私密。该处理不扣除旧版角色卡已获得的收益电量，也不影响已开始游玩的用户继续游玩；作者仍可通过主动分享旧版私密链接供他人访问体验。
- 转载：角色卡标记为转载属性，并按规则标注原作者信息；转载卡不会产生收益电量。
- 重新公开：经平台要求整改的角色卡，作者完成相应整改并经审核组确认后，可由私密状态恢复为公开状态。除本细则另有规定外，整改未完成或未经确认前，角色卡不得重新公开。

1.4　裁量与证据原则
- 平台在接到用户举报后，将结合查重结果与证据链进行综合判断。若违规行为对社区造成明显负面影响，如引战、骂战、攻击性争论持续发酵等，平台可按更高档处理执行。
- 平台在具体处理时，可结合以下因素进行从轻或从重判断：是否首次违规、是否主动说明与配合核实、是否及时整改、是否存在主观规避、是否造成社区争议、是否存在多账号协同、是否删除证据或干扰调查、是否存在反复违规或组织化行为。

1.5　违规处理的时效
平台对违规行为的处理不设追溯期限。无论角色卡发布时间长短、违规行为发生多久，一经查实，平台均有权依据本细则进行处理。

1.6　审核通过的效力范围
- 角色卡通过日常审核，仅代表该卡在提交审核时的公开展示内容符合当时的基本投递要求，不构成平台对该卡全部内容的合规性背书，亦不代表该卡在后续公开期间的内容及相关行为永久符合平台规则。
- 日常审核的主要工作为对新提交卡件进行基础筛查，审核资源有限，无法对每张角色卡的全部内容、后续更新进行实时全量核查。审核通过后，若角色卡出现新增内容或其他后续行为触发违规情形，或经举报、复查发现原有内容存在违规问题，平台仍有权依据本细则进行处理。
- 作者对其角色卡全部公开内容及后续公开行为负有持续的合规责任，不得以"已通过审核""平台此前未作处理"等理由作为免责依据。

1.7　整改复审程序
- 角色卡因违规被设为私密并经督促整改的，作者完成整改后可向审核组提交复审申请。
- 审核组将在收到复审申请后 6 至 72 小时内作出处理结果，具体时长视角色卡情况而定。
- 复审结果分为两种：确认整改完成，角色卡恢复公开；或整改仍不符合要求，继续设为私密并告知作者需进一步修改的具体问题。
- 若复审结果为继续私密，作者可在再次整改完成后重新提交复审申请，复审时限同上。

第二章　内容类违规

2.1　重复投递（相同角色卡重复上传）
触发条件
- 已公开的角色卡再次以新公开卡形式发布，且内容实质相同。
- 仅更换封面、修改标题、微调排版或少量措辞修改的，不构成"新卡"，原则上认定为重复投递。
处理方式
- 将新公开且被判定为重复的角色卡设为私密。
- 扣除该新卡的全部收益电量。

2.2　不符合重发更新条件（少量改动重发）
触发条件
- 角色卡仅进行轻微调整后重新作为新公开卡投递，且改动不足以构成"新卡"或"有效更新"。
- 轻微调整包括但不限于替换部分词句、人设小幅修改等。
- 尤其在角色卡人设字数尚未达到上限、可通过原卡继续完善的情况下，作者应优先在原卡更新，而非另行作为新公开卡发布。
处理方式
- 将该新公开卡设为私密。
- 扣除该新卡的全部收益电量。
- 要求作者在原卡更新。

2.3　有效更新与去旧留新
适用条件（满足其一即可纳入评估）
- 世界观大幅扩展。
- 角色人设大幅补充。
- 剧情结构 / 交互机制出现实质调整。
- 其他足以构成"版本跃迁"的实质内容变化，以平台综合判断为准。
处理方式
若经查重与综合判断符合"有效更新"，采取"去旧留新"处理：
- 将旧卡设为私密。
- 不扣除旧卡任何收益电量。
- 不影响已游玩玩家继续游玩。
- 作者仍可通过主动分享旧卡私密链接供他人访问体验。

2.4　敏感内容未投递至正确分区
触发条件
- 角色卡内容包含敏感元素，但作者未投递至正确分区。
处理方式
档 A（一般情况）
- 将该卡设为私密。
- 督促作者调整至正确分区，整改完成后方可重新公开。
档 B（造成明显社区影响，如引发大量争论、骂战、攻击、集中举报等）
- 将该卡设为私密。
- 督促作者调整至正确分区，并补充相应风险提示，整改完成后方可重新公开。
- 扣除该卡 50% 的收益电量。
注：若作者已通过标签、简介、第一句话、评论区置顶或其他显著方式，对相关敏感元素作出明确提示，平台可酌情优先按档 A 处理，但仍需调整至正确分区。

2.5　虚假包装 / 误导性投递
触发条件
- 角色卡的标题、简介、第一句话、封面、分区、标签或其他公开展示信息，与玩家实际核心体验、主要互动逻辑、剧情走向存在明显不符，足以对玩家形成错误预期。
- 故意弱化、隐藏或回避会显著影响玩家体验判断的关键要素（如主控受辱、背叛 / NTR 倾向、明显负向互动结构等），使玩家在进入游玩前无法作出基本判断。
- 以"纯爱 / 无牛"等包装高争议、高压、强负面体验内容，或通过模糊描述、误导性文案刻意误导玩家。
- 其他经平台综合判断，属于对作品核心体验进行明显包装性误导、足以造成用户混淆或误入的情形。
处理方式
档 A（一般情况）
- 将该卡设为私密。
- 督促作者补齐相应分区、风险提示，或修改标题、简介、第一句话、分区等公开展示内容，使其与实际核心体验相符，整改完成后方可重新公开。
档 B（造成明显社区影响）
- 将该卡设为私密。
- 督促作者修改标题、简介、第一句话、分区、标签等公开展示内容，使其与实际核心体验相符，并补充相应风险提示，整改完成后方可重新公开。
- 扣除该卡 50% 的收益电量。
注：若角色卡同时存在"敏感内容未投递至正确分区"（第 2.4 条）与"虚假包装 / 误导性投递"（第 2.5 条）的情形，平台可根据误导程度、用户与社区影响及作者主观规避迹象，优先适用第 2.5 条或从重处理。

2.6　角色年龄数字限制
触发条件
- 角色卡的标题、简介、第一句话或开场白中出现未满 18 岁的具体年龄数字。
处理方式
- 将该卡设为私密。
- 督促作者移除或调整相关年龄数字描述，整改完成并经审核组确认后方可重新公开。
注：用于追忆性、过去式描述的年龄数字（如"她在十七岁那年……"等回忆性表述），可结合上下文酌情评估，不当然视为违规。

2.7　幼态主题肉向角色卡的成年声明要求
触发条件
- 涉及幼态主题、幼态外形描写或包含幼态相关图片元素的角色卡，并且属于开局即涩类肉向卡（开场或前期即进入性互动 / 性场景，非剧情向慢热推进），未在标题或简介开头明确标注"所有角色均已成年"的成年声明。
处理方式
- 将该卡设为私密。
- 督促作者补充成年声明，整改完成并经审核组确认后方可重新公开。
注：偏剧情向、需要通过剧情发展逐步进入亲密互动的幼态角色卡（如公路卡、恋爱攻略、慢热剧情卡等），不强制要求添加成年声明。

2.8　涉及中国近现代敏感题材或真实人物
触发条件
- 角色卡内容涉及中国近现代敏感题材、敏感历史事件或相关敏感人物，未进行架空 / 魔改处理。
- 角色卡使用真实明星、艺人、网红、公众人物的真实姓名作为角色名或核心设定，未进行化名或架空处理。
处理方式
- 将该卡设为私密。
- 督促作者对相关题材进行架空、魔改、化名或其他规避处理，整改完成并经审核组确认后方可重新公开。

2.9　角色卡图片合规要求
触发条件
- 角色卡封面、第一句话配图、状态栏图片或其他作品内嵌图片包含真人露脸图像。
- 上述图片包含明显水印。
- 上述图片包含 AI 生成的过于逼真的真人图像。
- 上述图片包含过于幼态的图像。
处理方式
- 将该卡设为私密。
- 督促作者对相关图片进行打码、更换或去除水印等处理，整改完成并经审核组确认后方可重新公开。

2.10　违规引流
触发条件
- 角色卡的标题、简介、第一句话、开场白、设定、封面、状态栏、美化内容或其他任何位置，出现指向非官方平台或非官方社群的引流信息。
- 引流形式包括但不限于：其他平台 / 网站名称或链接、QQ 群 / 微信群 / TG / DC 等其他社群信息、个人社交账号、二维码、邀请码、ID 或引导关注的账号信息等。
- 涉及招嫖、约炮、付费色情服务、线下性交易引流，或其他违法 / 严重违规性质的引流内容。
处理方式
档 A（一般引流）
- 将该卡设为私密。
- 督促作者移除相关引流内容，整改完成并经审核组确认后方可重新公开。
档 B（多次督促不改或再次违规）
- 经多次督促仍未整改、整改后再次出现引流内容，或同一作者反复触犯本条款的，视情节严重程度封禁相关账号的投递权限。
档 C（招嫖、约炮等违法 / 严重违规引流）
- 将该卡设为私密。
- 扣除该卡的全部收益电量。
- 直接封禁相关账号的投递权限。
注：是否构成"引流"，以平台基于内容指向、表达意图及实际效果的综合判断为准；正常的世界观引用、致敬类标注（如"参考自 XXX 作品"）不当然视为引流。

第三章　原创性违规

3.1　抄袭 / 搬运 / 重复他人作品
触发条件
- 非原创却标记原创。
- 盗用他人作品。
- 复制其他作者角色卡设定、结构或大量文本。
- 未经授权直接搬运他人作品。
- 其他经平台综合判断，属于抄袭、搬运或重复他人作品的情形。
处理方式
- 将该角色卡设为私密。
- 扣除该卡的全部收益电量。
注
- 平台判断抄袭 / 搬运时，将综合参考文本重复率、内容相似度，以及核心设定、人物关系、剧情结构、关键事件链、特殊机制、关键表达段落、表达结构与整体体验是否存在高度重合。
- 重复率与相似度是重要参考因素，但平台将优先关注有效内容层面的实质重合，而非仅以字面重复比例作为唯一依据；仅题材、分区、常见元素相同，不当然构成抄袭。
- 本条款的累犯加重适用第 5.1 条。

3.2　第一句话抄袭他人作品
触发条件
- 第一句话直接复制粘贴他人小说 / 文本内容，或照搬多段且未做任何实质改写润色。
处理方式
- 将该角色卡设为私密。
- 扣除该卡 50% 的收益电量。
- 督促作者更改并补充引用来源，整改完成后方可重新公开。
统一引用格式（模板）
- 创作声明：本角色卡的第一句话 / 开场白部分内容引用自 XXX（作品名称），作者 XXX。

3.3　高度借鉴他人作品但未做引用
触发条件
- 角色卡核心设定 / 关键结构明显参考他人作品或角色卡，但未作引用声明，造成原创误导。
处理方式
- 将该卡设为私密。
- 扣除该卡 25% 的收益电量。
- 作者需在以下位置任选其一添加参考来源后方可重新公开：简介 / 第一句话 / 开场白 / 评论区置顶。
统一引用格式（模板）
- 创作声明：本角色卡的核心设定 / 关键结构参考自 XXX 角色卡（角色卡名字），作者 XXX。
注：若已引用仍存在实质抄袭 / 搬运，按第 3.1 条处理。

3.4　美化 / 状态栏模板复制挪用
触发条件
- 角色卡的美化内容，如状态栏、按钮样式、排版框架、HTML / CSS 美化组件等，存在未经授权的直接复制挪用他人作品，且未做任何实质性修改。
处理方式
- 将该卡设为私密。
- 扣除该卡 30% 的收益电量。
- 督促作者修改 / 替换；如涉及参考他人模板，需按平台要求补充来源 / 授权说明，整改完成后方可重新公开。
统一引用格式（模板）
- 创作声明：本角色卡的状态栏 / 美化样式 / HTML 排版 / 按钮样式参考或使用了 XXX 模板 / 角色卡的美化结构，作者 XXX。

第四章　账号与行为类违规

4.1　非作者本人原创发卡（代发 / 买卡 / 冒名投稿）
触发条件
- 投稿账号并非作品实际作者，且存在代发、买卡、冒名投稿等情况。
- 包括但不限于未经实际作者本人投稿、以金钱交易方式取得作品后由他人账号发布、冒用他人身份投稿等情形。
处理方式
- 将该角色卡设为私密。
- 扣除该卡的全部收益电量。
注
- 不得以"双方协商一致"或"存在金钱交易"作为免责理由。
- 经原作者明确授权的转载、协助整理或代为上传，不当然等同于原创投稿；发布账号不得将该作品标记为本人原创或获取原创收益，具体应按转载、换绑或平台认可的授权发布方式处理。
- 涉及授权代为上传的，作者与发布账号应保留授权凭证；如平台要求核实，应配合提供。

4.2　多号发卡
触发条件
- 同一作者使用多个账号发布角色卡、获取角色卡收益电量，或通过多个账号伪装为不同创作者身份，制造多个独立作者、不同作品来源或不同社群影响力的假象，以达到投稿、收益获取、引流或规避平台"一名作者仅可使用一个账号发卡"管理规则的目的。
- 平台将根据账号注册信息、登录 IP、作品风格、账号关联性、投稿记录、收益获取情况及其他后台数据，综合判断多个账号是否实际归属于同一作者或受同一作者控制。
处理方式
- 将相关账号名下所有角色卡全部设为私密。
- 扣除相关角色卡的全部收益电量。
- 封禁相关账号的投递权限。
- 仅保留最早注册的账号，其余账号按规则处置。
- 情节严重、存在反复规避、收益转移、虚假身份说明或拒不配合核实的，平台可进一步限制相关账号其他权限。
注
- 平台不允许同一作者通过多个账号分别发卡。
- 是否构成多号发卡，以平台基于账号关联性、投稿行为、作品风格及证据链的综合判断为准。
- 作者不得以"账号是朋友的""只是帮忙发布"等理由规避责任。若多个账号实际服务于同一作者发卡、收益获取或规则规避目的，平台可按行为实质进行认定。

4.3　刷对话量 / 刷收益电量
触发条件
- 角色卡作者本人账号，或与作者存在关联关系、利益关系、协同行为的账号，通过本人账号、小号、关联账号、互刷账号、代刷账号或其他方式，对指定角色卡进行异常对话量、异常电量消耗、异常收益制造，或刻意制造热度数据、榜单表现及其他非自然曝光优势。
- 相关行为足以影响角色卡自然流量增长、收益公平、榜单秩序或社区公平竞争环境。
- 平台将结合有关账号的电量消耗记录、操作时间间隔、聊天内容、账号关联性、行为集中度及其他后台数据进行综合核实与判断。
处理方式
- 将涉及刷对话量 / 刷收益电量的角色卡设为私密。
- 扣除该角色卡的全部收益电量。
- 在扣除该角色卡收益电量之外，额外扣除违规账号当前账号余额电量的 45%；按 45% 计算后不足 10 万电量的，按 10 万电量扣除。扣除电量超出当前余额的，差额计为账号负余额。
- 情节严重、存在多账号协同、组织化刷量、反复违规或规避处理的，平台可参照第 4.2 条"多号发卡"及第 8.1 条"未尽事项与补充处理"从重处理。
注
- 是否构成刷量，以平台基于后台数据、行为模式及证据链的综合判断为准。正常用户游玩、自然传播、作者正常测试角色卡，不当然视为刷量；但若测试行为明显超出合理频率、持续时间、消耗规模，或与收益获取、榜单冲刺、异常曝光结果存在明显关联，平台仍可按刷量行为综合认定。
- 作者不得以"账号并非本人操作""朋友自愿支持""只是测试角色卡"等理由规避责任。若相关账号与作者存在明显关联、协同行为或收益指向关系，平台可按行为实质进行认定。

4.4　虚假举报与干扰审核
触发条件
- 提交明显与事实不符、伪造证据或恶意针对特定作者的举报。
- 自导自演、组织化举报、利用举报机制赚取奖励或干扰他人作品分发。
- 删除证据、误导调查、串供、阻碍审核组核实工作的其他行为。
处理方式
- 视情节轻重，回收相关奖励、封禁举报权限。
- 情节严重的，封禁投递权限，并按本细则其他相关条款合并处理。
注：因虚假举报导致他人角色卡被错误处理的，平台应当及时撤销原处理，恢复角色卡公开状态，并返还已扣除的收益电量。

第五章　累犯与处罚竞合

5.1　累犯加重
同一作者累计 ≥3 次触犯第 3.1 条（抄袭 / 搬运）的，除按该条款处理外，另行：
- 封禁投递权限。
- 扣除该作者名下所有角色卡的全部收益电量。
其他条款下的累犯，平台可参照本条精神，结合违规性质、频次、主观恶意及社区影响，从重处理。

5.2　处罚竞合
- 同一角色卡或同一账号同时触犯多项违规条款的，平台可根据行为性质、违规目的、实际影响及证据链，选择适用较重条款，或对不同违规行为分别处理、合并执行。
- 涉及收益、电量、投递权限等处罚的，平台可按实际违规所得、影响范围及从重情节进行综合扣除。

第六章　举报

6.1　举报渠道
用户可前往平台官方 TG / DC 社群的"举报违规角色卡"专区进行举报，将相关举报信息发送至该社群即可。

6.2　举报信息
举报时应提供以下内容：
- 被举报角色卡的身份信息——可直接发送角色卡截图（推荐），或文字提供角色卡的链接、ID、名称等。
- 涉嫌违反的具体条款或违规情形的简要描述。
注：审核组将依据举报者提供的信息定位角色卡并核实违规内容；若有补充材料（如原始来源链接、对比截图等），可一并发送以协助核实。

6.3　举报处理
- 审核组在收到举报后将进行核实与判断，并根据本细则相关条款对违规角色卡作出处理。
- 核实结果及处理情况由审核组依规执行；举报者可在合理期限内向审核组询问处理进度。

6.4　举报奖励
- 经审核组核实属实的违规角色卡，举报者将获得 200 电量的奖励，每张违规角色卡对应 200 电量。
- 同一张角色卡被多人举报且均属实的，奖励发放给最早提交有效举报的用户；其他举报者不重复发放。
- 举报内容经核实不属实、证据不足或存在虚假举报情形的，不发放奖励；情节严重的按第 4.4 条处理。

第七章　申诉

7.1　申诉渠道
若用户对判定结果有异议，可通过 TG / DC 官群联系审核组进行申诉。

7.2　申诉时限
申诉应自处罚通知送达之日起 7 日内提出。逾期未提出申诉的，原处理结果生效。

7.3　申诉材料
申诉时应提供以下材料：
- 申诉账号信息及涉及的角色卡身份信息——可直接发送角色卡截图（推荐），或文字提供账号 ID、角色卡链接 / ID / 名称等。
- 对判定结果的具体异议及理由。
- 支持申诉理由的证据材料（如原创证明、授权凭证、聊天记录等）。
注：审核组将依据可获取的信息进行核实；若信息不足以支持申诉判断，审核组可要求申诉者补充材料。

7.4　申诉处理
- 审核组将基于既有证据与判定依据进行一对一回复，回复时限原则上为 3 日内。
- 申诉结果为维持原处理或调整处理的，审核组应说明依据。
- 对申诉结果仍有异议的，作者可在收到答复后 3 日内提出二次申诉，审核组将启动同行评审机制进行综合裁定，由非原审理成员组成的复核小组进行评议，以确保处理结果公正一致，复核结果为最终结果。

第八章　附则

8.1　未尽事项与补充处理
- 对本细则未明确列举，但实质上破坏平台创作秩序、发布秩序、社区公平、内容合规或用户体验的行为，平台有权结合规则目的、证据情况及实际影响进行综合判断，并参照最相近条款处理。
- 若用户通过规避、拆分、变形等方式实施与本细则所禁止行为实质相同的行为，平台有权按照行为实质进行认定与处理，不以表面形式差异作为免责依据。

8.2　规则解释与适用
- 本细则由平台制定，具体执行、适用说明及申诉受理由审核组负责。
- 在具体执行中，平台将结合规则文本、证据链、行为性质、实际影响及历史处理口径进行综合判断；对于存在争议或规则边界不明的情形，平台保留进一步说明、补充规则适用口径及调整处理尺度的权利。
- 本细则的最终解释权归平台所有。

8.3　修订与公示
本细则的修订由平台发起，修订后将在官方渠道公示，自公示之日起立即生效，修订前已生效的处理结果不受影响。
