# 状态栏规则 · 修仙卡-影渲美化 v2（当前 MMD / 影渲法 ShadowCast）

本文件是**模型侧协议** + **导入说明** + **自定义指南**。
把「状态面板生成协议」一节作为一条 **constant=true（蓝灯/常驻）世界书条目**放进卡里，模型才会每轮持续输出 `<xx>` 数据块，状态栏才会更新。json 里的正则只负责**渲染**。

---

## 一、导入说明

`output/修仙卡-影渲美化.mmd.json` 为 MMD 四字段导入格式，含 4 条正则：

| 正则 | 触发标记 | 作用 |
|---|---|---|
| 水墨全局美化 | `<美化>` | 全局换肤（穿透方案，气泡/输入框/弹窗/图标染色） |
| 修仙状态栏渲染 | `<xx>…</xx>` | 消息末尾交互状态栏（影渲法 Shadow DOM） |
| 悬浮球-shadow | `<悬浮球>` | ☯ 可拖动球 + 菜单（影渲法，挂 body 常驻） |
| 侧边栏-shadow | `<侧边栏>` | 多窗口侧边栏：舆图/图鉴/注入（影渲法，挂 body 常驻） |

触发标记由**开场白**插入（json 的 beginning 已含全部标记 + 一份演示态全量数据，导入即生效）。悬浮球/侧边栏单例防重，reload 后再发消息自动重建。

## 二、状态面板生成协议（放入卡内蓝灯常驻条目 · 通用模板）

> **本协议通用于任意修仙角色卡。下方 `{大括号}` 为占位符，请一律按【当前这张卡的实际角色/世界设定】填写**——角色名、灵根、功法、地名、NPC 等全部取自当前剧情，**严禁照抄本协议里的任何示例词**（示例中的人名/功法名/地名只为说明格式，不属于任何具体角色）。**只保留键名与分隔结构，值全部换成当前卡的真实内容。**

每次回复正文结束后**另起一行**输出当前状态。固态字段每轮必出（即使没变也照原值重出）；情境字段仅在对应情境那轮输出。**格式（占位符版，按当前角色填充）**：

```
<xx>
name={角色名}|{性别}|{年龄}|{灵根名}|{天/上/中/下品}|{灵根特性描述};loc={大区域}-{地区}-{当前详细地点};realm={当前境界}|{当前修为值}/{该境界上限};gongfa={心法或功法}|{功法名}|{当前程度}|{当前/上限}|{总层数或圆满}|{各层效果}|{功法描述};dress={衣物/法器名}|{描述},{衣物2}|{描述2};skill={剑诀/术法/神通名}|{说明},{技法2}|{说明2};bag={物品名×数量}|{描述},{物品2}|{描述2};npc={在场角色名}|{外貌}|{我对其印象与实力猜测},{角色2}|{外貌2}|{印象2};opts={激进行动},{中庸行动},{保守行动},{暧昧/情欲试探行动}
</xx>
```

### 三级分隔符（务必遵守）

| 层级 | 符号 | 用途 |
|---|---|---|
| 字段间 | `;` 半角分号 | 顶层分隔不同字段 |
| 同字段多条目 | `,` 半角逗号 | 着装多件、在场多人 |
| 条目内部件 | `\|` 竖线 | 名\|描述、名\|外貌\|印象 |

> **描述正文一律用全角标点**（，。、；：），半角 `, ; | = < >` 是保留分隔符，写进描述会破坏解析。中文写作天然用全角，几乎无感。

### 字段表（格式 + 占位模板）

| 键名 | 含义 | 格式 | 占位模板（按当前角色填） | 代谢 |
|---|---|---|---|---|
| `name` | 姓名 | 名\|性别\|年龄\|灵根\|品级\|灵根描述 | `{角色名}\|{性别}\|{年龄}\|{灵根名}\|{天/上/中/下品}\|{灵根特性描述}` | 固态（继承） |
| `loc` | 方位 | 大区-地区-详细地区 | `{大区域}-{地区}-{当前详细地点}` | 固态（继承） |
| `realm` | 修为 | 境界名\|当前/最大 | `{当前境界}\|{当前修为值}/{该境界上限}` | 固态（继承） |
| `gongfa` | 功法 | 类别\|名\|程度\|当前/最大\|总层或圆满\|各层效果\|描述 | `{心法或功法}\|{功法名}\|{当前程度}\|{当前/上限}\|{总层数或圆满}\|{各层效果}\|{功法描述}` | 固态（继承） |
| `dress` | 着装 | 名\|描述,名2\|描述2 | `{衣物/法器名}\|{描述},{衣物2}\|{描述2}` | 固态（继承） |
| `skill` | 技法 | 名\|说明,名2\|说明2 | `{剑诀/术法/神通名}\|{说明},{技法2}\|{说明2}` | 固态（继承） |
| `bag` | 乾坤袋 | 名\|描述,名2\|描述2 | `{物品名×数量}\|{描述},{物品2}\|{描述2}` | 固态（继承） |
| `npc` | 在场人物 | 名\|外貌\|印象与实力猜测 | `{在场角色名}\|{外貌}\|{我对其印象与实力猜测},{角色2}\|{外貌2}\|{印象2}` | 固态（继承） |
| `trial` | 渡劫·斗法 | 对象\|状态说明 | `{劫/敌名}\|{状态说明}` | **情境（用完即焚）** |
| `opts` | 行动 | 激进,中庸,保守,暧昧（固定四项按序） | `{激进行动},{中庸行动},{保守行动},{暧昧/情欲试探行动}` | **每轮新生成（不继承）** |

### 各字段写法要点（结构说明，占位符按当前卡替换）
- `name` 姓名：年龄显小字；灵根做成**点击气泡**「品级（居中）—分隔线—灵根描述」。品级用 天/上/中/下品（或本卡设定）。
- `loc` 方位：三段用半角 `-`，地名取自当前世界观。
- `realm` 修为：`境界名|当前/最大`，境界体系按本卡设定。
- `gongfa` 功法（**修炼/淬体类，可多门**）：类别取 `心法`/`功法`（名前标签）。每门带进度条+数字，名字点击弹气泡「总层或圆满（居中）/各层效果/分隔线/描述」。**攻伐剑诀归技法，不放这里**。
- `skill` 技法：放**临战施展的剑诀/术法/神通**，点击弹说明气泡。
- `dress` 着装 / `bag` 乾坤袋：`名|描述` 逗号分隔，点击弹描述气泡；乾坤袋**点标题展开**。
- `npc` 在场人物：`名|外貌|印象与实力猜测`，**只写外貌和『我』的主观印象**，不写对方真实数值。
- `trial` 渡劫·斗法：**仅渡劫/斗法那轮输出**，情境结束就停。
- `opts` 行动（**每轮必出，最重要**）：固定四项按序 激进/中庸/保守/暧昧，第四项暧昧/情欲（绯色♥）。每项一句第一人称玩家行动指令，点击回填输入框。每轮按当前情境生成。

### 铁律
- 用 `<xx>` 包裹，字段间 `;`，键值 `=`。推荐标准键名（引擎对别名归一化容错，如写 `境界` 归到 `realm`）。
- **所有值按当前角色卡实际填写，不沿用本协议示例里的人名/功法名/地名**（防跨卡串台）。
- 固态字段每轮必出；情境字段 `trial` 只在情境那轮出；`opts` 每轮按情境新生成。
- 描述正文用全角标点；不要出现半角 `; | = < >` 和换行。
- 不解释、不加排版标记，只吐数据块。

> 完整具体示例（**仅演示格式，禁止照抄词句**）见 `工作/状态栏协议.md` 末尾，或参考 json 开场白里的演示数据。

## 三、组件说明（无需写进卡，给你看）

**悬浮球 ☯**（左下，可拖动，hover 太极反色）点击展开菜单：
- ✍ 字数设定 → 选预设字数 → 回填"请将本次回复字数控制在约N字"到输入框
- 🔄 状态栏回复 → 回填刷新状态栏指令
- 📖 游玩指令 → 弹玩法帮助气泡

**侧边栏**（右侧贴边按钮，hover 变朱砂展开），点按钮弹窗：
- 🗺 舆图：底图 + 可点地标（青冥宗/天南坊市/万兽山脉/东海之滨），点地标在下方显示该地说明
- 👥 图鉴：固定名单（云清/玄阳子/血煞老祖），两列竖图，**点卡片图片模糊+叠加介绍**
- 💉 注入：开关 + 文本框，开启后每次发送自动在消息尾部附加文本框内容（见下"自动注入说明"）

### ⚠️ 自动注入说明（重要）
- 主方案：hook 发送按钮/回车的捕获阶段，发送前把注入文本追加进输入框。
- **真机能否拦截发送取决于 MMD 当前实现**（属未在真机验证项）。若拦截失败，开关开启时会退化为"立即把内容回填到输入框"，你手动确认后发送。
- 导入后请实测一次：开开关→输入消息→发送，看尾部是否带上注入内容。

## 四、自定义指南（你后续要改的）

源生成器在 `工作/`，改完重新生成 + 重新合并：
- **状态栏字段/配色** → `build_xianstatusbar.py` 的 `FIELDS` / `SHADOW_CSS`
- **悬浮球菜单/字数预设/帮助文本** → `build_xianfloat.py` 的 `MENU_ITEMS` / `WORDCOUNTS` / `HELP_TEXT`
- **地图底图与地标** → `LANDMARKS`（x/y 是百分比坐标）/ `MAP_IMG`（换成你的真实地图图片 url）
- **角色图鉴名单与竖图** → `GALLERY`（把 `https://xxx.xx/char_*.jpg` 换成真实竖图链接，改 name/intro）
- **自动注入默认文本** → `INJECT_DEFAULT`

风格 token：宣纸 #f4f0e8 / 淡墨 #3a3632 / 朱砂 #a85545 / 黛青 #5a6b6e / 竹青 #5a7d52。圆角 10px，宋体优先。
