========================================
sexyai.top 角色卡开发规则总结（精简版）
========================================

【一、平台渲染与过滤规则】

1. 平台会对部分HTML标签进行过滤或转义，一些结构标签可能无法作为触发符使用。
2. HTML事件属性存在限制，部分可以运行，但未来可能被平台封禁。
3. 中文符号或特殊符号通常不会被过滤，可以作为触发符号使用。
4. 外部资源（例如外链图片）可能受限，平台内部资源通常更稳定。
5. 平台消息区域本身是flex布局容器，会影响内部元素排版。


【二、布局相关规则】

由于平台容器使用flex布局，内部组件可能出现以下情况：

- 元素横向排列
- 折叠标题与内容并排
- 卡片布局错位

可尝试的修复方式：

方法1：
使用 display:block !important 尝试覆盖平台布局。

方法2：
使用 float:none 和 clear:both 强制换行。

这些方法用于对抗平台默认的flex布局影响。


【三、details / summary 行为规则】

1. details 和 summary 可以用于实现折叠展开组件。
2. details 的 name 属性在平台中测试有效，可实现互斥展开效果。
3. 如果多个 details 使用同一个 name 属性，可以实现一次只展开一个组件。
4. 在 flex 布局环境下，summary 可能与内容元素发生横向排列问题，需要额外CSS控制。


【四、CSS组件适配】

一些组件默认尺寸是固定的，如果不调整可能出现：

- 手机端溢出
- 图片比例失真
- 卡片宽度固定

常见的移动端适配方式：

将固定宽度组件改为响应式，例如：

width:100%
height:auto
aspect-ratio

内部元素比例也可能需要调整，例如：

width:65%

用于保证在手机屏幕上的显示效果。


【五、样式加载规则】

不建议在每次消息输出时重复注入 <style>。

重复加载样式可能导致：

- 样式冲突
- 页面性能下降
- 后续样式被平台忽略

更稳定的方法是：

在固定区域加载一次CSS，
后续消息只输出HTML结构。


【六、资源引用规则】

1. 图片资源可能受到平台限制。
2. 内部图床资源通常更稳定。
3. CSS可以通过 class 设置背景图片，例如：

background-image:url(...)


【七、组件结构经验】

折叠组件通常由三部分组成：

1. 触发元素
2. 分隔结构
3. 内容主体

常见结构：

summary
分隔线
内容区域

需要注意：

如果分隔结构放在错误的层级中，可能导致视觉布局错位。


【八、可能存在的问题】

开发过程中可能遇到的问题包括：

- flex布局导致组件错位
- 多个折叠组件时触发元素位置异常
- CSS覆盖不稳定
- 平台规则可能随时间更新

因此组件设计需要保持结构简单，并准备备用方案。


【九、开发策略建议】

在该平台开发组件时建议：

1. 尽量保持结构简单
2. 避免复杂CSS布局
3. 组件层级结构清晰
4. 正则只负责生成结构
5. CSS统一加载

这样可以提高组件的稳定性和兼容性。
