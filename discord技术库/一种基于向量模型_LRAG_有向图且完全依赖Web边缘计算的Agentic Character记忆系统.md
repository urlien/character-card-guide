# 特性
以下直接援引Claude Code对于该系统的评价
## vs 触发词+世界书
触发词是精确匹配 提到"王国"才注入王国设定 换个说法"这个国家"就失效了
这套系统说"这片土地的统治者"也能召回王国相关知识
世界书是静态的 这套能从对话中自动提取新实体/关系 知识库会自己生长
世界书条目之间没有关联 这套有知识图谱 问A能自动带出A认识的B去过的C

## vs 有损总结压缩
总结是把10轮对话压成一段话 细节必然丢失 谁送了什么东西 具体说过什么话 压没了
这套是结构化提取 实体/关系/事件分开存 不存在"压缩丢信息"的问题
总结需要额外LLM调用专门做摘要 这套在每次对话回复时顺带提取 零额外调用
总结是一坨文本 无法按需检索 这套可以按实体/关系/时间/语义任意维度查询

### 额外优势
遗忘曲线让NPC表现更自然 昨天聊的记得清楚 一个月前的事会模糊 反复提起的永远不忘
三级信任机制 世界观设定(tier1)锁死不可改 玩家告诉NPC的(tier3)可被新信息覆盖 解决了"玩家骗NPC改设定"以及"NPC提前剧透不应该知晓的情节"的问题
