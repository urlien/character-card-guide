## 能实现的操作

### 基础能力

- **编写网页代码**：可以自由编写 HTML、CSS、JavaScript
- **画图和动画**：可以用 Canvas 画布做绑制、做动画效果
- **3D 图形**：可以使用 WebGL 创建 3D 场景
- **音视频播放**：可以嵌入和播放音频、视频
- **运行 WASM**：可以加载并运行 WebAssembly 程序
- **提交表单**：可以创建表单并提交
- **动态脚本**：可以使用 `eval()` 动态执行代码
- **后台任务**：可以用 Web Worker 在后台运行脚本

### 数据存储

`Bridge` 通过劫持原生接口 你可以：

| 存储类型 | 说明 |
|---------|------|
| localStorage | 简单的键值对存储，适合存少量文本 |
| IndexedDB | 结构化数据库，适合存大量数据 |
| OPFS 文件系统 | 私有文件存储，适合存文件内容 |

**注意**：每个博客的存储空间是隔离的，你只能访问自己的数据，看不到别人的。

### 网络请求

| 请求类型 | 说明 |
|---------|------|
| 外部请求 | 可以请求任意外部网址，但受浏览器跨域策略（CORS）限制 |
| 站内 API | 仅能调用该页面上位于白名单内的接口，其他接口会返回 404 |

---

## 受到限制的操作

### 安全限制

| 操作 | 原因 |
|------|------|
| 读写 Cookie | 完全禁止 |
| 操作主页面 | 无法访问主页面 |
| 调用非白名单 API | 会被拦截 |
| 页面跳转 | 无法让主页面跳转到其他网站 |
| 触发下载 | 无法让用户下载文件 |
| 访问剪贴板 | 无法读取或写入剪贴板 |
| 获取地理位置 | 无法获取用户位置信息 |
| 摄像头/麦克风 | 无法请求这些权限 |
| 屏幕共享 | 无法使用屏幕共享功能 |
| 弹出新窗口 | 默认禁止（除非特别开启） |

### 存储限制

| 操作 | 说明 |
|------|------|
| 直接用原生存储 | `localStorage`、`sessionStorage`、`indexedDB` 受到`Bridge`监管 |
| 访问他人数据 | 无法访问其他博客或其他用户的存储内容 |

---

## 数据保存情况

### 刷新后会丢失的数据

- JavaScript 变量里的数据
- 内存中的所有状态
- 页面上动态创建的元素
- `sessionStorage` 不可用
- Blob URL 创建的临时文件
- Service Worker（无法在页面注册）

**Nix**: 不不 不是这个意义上的白名单
这套系统将整个网页分为安全层和显示层 白名单位于安全层 指的是这个网页允许调用的同源接口列表(换句话说就是你不能在玩卡的界面修改自己的角色卡) 安全层整体对玩家是透明的
显示层由一个具有沙箱配置的全屏iframe承担 相当于dom主容器 玩家所有的操作以及可以看到的所有元素都在这里 这是一个和安全层解耦合 通过一套sdk(也就是`bridge`系统)进行沟通 无状态的显示容器
安全层在初始化这个iframe时会劫持一系列的函数(大部分是将原有的直接调度分配给bridge系统 并非出于安全考量 否则玩家的所有操作都不会被持久化储存) 然后再初始化玩家代码
而在iframe里你是可以自由调用所有外域url的 比如从github获取代码 执行js甚至wasm都是完全自由的
但是如果需要调用同源的接口 则会被劫持的fetch直接上送给bridge系统 最终由安全系统审核之后 决定是否放行
这一切都是自动的
